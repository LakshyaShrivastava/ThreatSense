# GitHub repository metadata

GitHub pin cards use **Description** and **Topics** from **Settings → General** (not from files). Apply these there so your profile pins stay readable.

| Field | Value |
|--------|--------|
| **Description** | Reactive pipeline demo: synthetic network logs → Kafka → MongoDB via Connect; Spring Boot WebFlux consumer reads change streams and persists structured logs + alerts to PostgreSQL. |
| **Topics** | `java`, `spring-boot`, `kafka`, `mongodb`, `postgresql`, `docker`, `spring-kafka`, `webflux`, `cybersecurity`, `logs` |

Copy topics one at a time into the Topics field (GitHub suggests tags as you type).
